# **Lógica de Programação**

Repositório destinado às atividades desenvolvidas na disciplina de : Lógica de Programação - 1Sem / 2026 - Tecnologia de Ciência de Dados

## Estrutura

**- 01 - Exercícios de fixação e aprendizado - L1**
    - Exercício 1 - Operadores Aritméticos
    - Exercício 2 - Operadores Atribuição
    - Exercício 3 - Operadores Comparação
    - Exercício 4 - Operadores Lógicos
    - Exercício 5 - Operadores de Identidade (is, is not)
    - Exercício 6 - Operadores de Associação (in, not in) 
    - Exercício 7 - Estrutura Condicional (IF, ELIF, ELSE)
    - Exercício 8 - Estrutura de Repetição FOR
    - Exercício 9 - Estrutura de Repetição WHILE

**- 02 - Exercícios de fixação e aprendizado - L2**
    - Exercício 1 - Integração: Operadores + Condicionais
    - Exercício 2 - Integração: Repetição FOR + Operadores de Associação
    - Exercício 3 - Integração: Repetição WHILE + Condicionais + Atribuição
    - Exercício 4 - Integração: Identidade + Coleções Simples
 
**- 03 - Exercícios de fixação e aprendizado - L3**
    - Case: Você recebeu uma lista de transações de uma empresa. Seu trabalho é
      criar um script que processe essas transações, verifique a integridade dos dados e
      calcule o saldo final, mas com regras rígidas de segurança.

**- 04 - Desafio estudo de caso: A Crise da Startup TechInvest**
    - Case fictício da startup TechInvest, que precisa criar o “Prime”, um robô-analista em Python para atender clientes e orientar investimentos sem uso de banco de dados.
      O programa coleta dados financeiros, analisa a situação do usuário, identifica seu perfil de risco e sugere investimentos com base em regras condicionais.
      Também projeta objetivos futuros considerando disciplina e tempo de investimento.

**- 05 - Desafio: Processamento de Dados de Vendas**
    - Processar dados de vendas inseridos pelo usuario, aplicando os conceitos de Listas e Tuplas em Python
 
**- 06 - Desafio Pipeline de Auditoria para Ciência de Dados**
    - Case de Ciência de Dados para desenvolver um pipeline de auditoria que coleta, limpa e analisa vendas de filiais com dados inconsistentes.
      O sistema utiliza funções, loops e condicionais para filtrar valores inválidos, calcular faturamento total e média, e classificar a performance da filial.
      Ao final, gera um relatório formatado com métricas consolidadas e listagem das vendas válidas.
      Requisitos técnicos utilizados: Funções; Estrutura de Repetição (while); Estrutura de Repetição (for); Listas e Tuplas; Condicionais (if/elif/else); Operadores.

**- 07 - Exploração Do Planeta – Aplicação De Função**
    - Case fictício de exploração espacial em Alphara-7, onde a lógica do programa simula decisões da tripulação diante de obstáculos e descobertas científicas.
      Utiliza funções, condicionais e repetições para controlar o fluxo da missão, incluindo tentativas de superar desafios e coleta de dados.
      Ao final, o sistema soma as conquistas obtidas e encerra a expedição com o retorno à nave.
